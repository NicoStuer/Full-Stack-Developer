Fundamentos JavaScript - Clase 2

Variable var Vs. let

  • No usar nunca la declaracion de una variable con var, dado q JavaScript las considera globales y deben estar muy seguros para usarla en todo el ambito del documento, a diferencia de let y const.

  • Var no respeta los scopes de block-variable global

  • Let genera variables de block

Variable const

  • No se puede inicializar vacia
  • Se utiliza para valores q no cambian dentro de el codigo
  • Las constantes se comportan de forma diferente entre valores Primitivos y valores Compuestos.

Funciones - Clase 7

Una funcion es un bloque de codigo, autocontenido, que se puede definir una vez y ejecutar en cualquier momento. Opcionalmente, una funcion puede aceptar parametros y devolver un valor.
Las funciones en JavaScript son objetos, un tipo especial de objetos:
Se dice que las funciones son ciudadanos de primera clase por que pueden asignarse a un valor, y pueden pasarse como argumentos y usarse como un valor de retorno.

Funcion Declarada



function estoEsUnaFuncion(){
   console.log("Uno");

   console.log("Tres");
}

Funcion con return

Para que una funcion nos retorne un valor se utiliza la palabra reservada "return" e inmediatamente la funcion deja de ejecutarse.

   return "vuelve este texto a pantalla"
}

Tipos de Operadores - Clase 10

Parametros Rest & Operador Spread

Parametros Rest

Son una forma de virtualmente ir agregando parametros infinitos, se declara anteponiendo tres puntos suspensivos"...".

function sumar(a, b, ...c){
    let resultado = a + b;
    c.forEach(function(n){
    resultado += n
)};
return resultado;
}


Spred Operator

Se utiliza para unificar arreglos.

const arr1 = [1, 2, 3, 4, 5],
   arr2 = [6, 7, 8, 9, 0];

const arr3 = [...arr1, ...arr2];

Arrow Functions

Es una forma de definir una funcion anonima.

const saludar = () => {
    console.log("hola");
}
saludar();

Cuando la funcion tiene una sola linea de instrucciones, se pueden omitir las llaves

const saludar = () => console.log("hola");


Los parametros de las funciones van dentro de los parametros.

const saludar = (nombre) => console.log(`hola ${nombre}`);

saludar("Nico");


Cuando una funcion flecha no recive parametros, indefectiblemente lleva los parentesis. Si la funcion recive solo un parametro pueden omitirse los mismos.

const saludar = nombre => console.log(`Hola ${nombre}`);
saludar("Nico");


Con las arrow function de una linea tambien podemos omitir el return:

const sumar = (a, b) => a + b;
